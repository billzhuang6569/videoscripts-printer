# Product

## Register

product

## Users

Production, creative, and AI workflow operators who need to turn structured shooting-script data into reliable printed tables. They use the app locally, usually while preparing a shoot, checking storyboards, or handing a printable PDF to a team.

## Product Purpose

HTMLprinter renders imported script-table JSON into an A4 print preview. It lets users adjust layout presentation only: field order, printed labels, visibility, column widths, row height, orientation, and shared templates. It does not edit imported script content. Success means a user can load a session, tune the print layout quickly, and print or save a clean PDF without touching the source data.

## Brand Personality

Calm, practical, precise. The interface should feel like a dependable production tool: quiet enough for repeated use, clear enough for non-technical teammates, and careful about print fidelity.

## Anti-references

Avoid marketing-page composition, decorative dashboards, oversized hero typography, glassmorphism, gradient text, loud color washes, nested cards, playful custom controls, and any UI that makes the print task feel secondary. Avoid content-editing affordances because imported script content is read-only in this app.

## Design Principles

- Print preview is the main product surface.
- Layout controls should be fast, direct, and reversible.
- Imported content stays read-only; presentation changes belong in templates.
- Use familiar product UI patterns so operators can trust the tool without learning a new mental model.
- Prioritize legibility, contrast, and predictable A4 output over visual novelty.

## Accessibility & Inclusion

Target WCAG AA for contrast, keyboard focus, semantic controls, and touch target size. Motion should be minimal and respect reduced-motion preferences. Chinese labels and long production terms must remain readable without clipping.
